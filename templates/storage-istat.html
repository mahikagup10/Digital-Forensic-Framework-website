View Metadata Structure

This tool uses the 'istat' plugin of The Sleuth Kit to display detailed information about metadata structures, such as an inode, within a file system. It allows forensic investigators to access and examine specific metadata associated with files, directories, or allocated space within a disk image or mounted file system. This tool provides essential insights into various attributes linked to an inode, including permissions, timestamps, file size, block allocation details, and other crucial metadata associated with files or directories.

FAQ

Common questions

Here are some of the most common questions that we get.
What is storage forensics? Storage forensics is the process of analyzing and extracting data from storage devices such as hard drives, solid-state drives, and USB drives to investigate and gather evidence for legal or investigative purposes.
What is memory forensics? Memory forensics involves the analysis and examination of a computer's volatile memory (RAM) to retrieve valuable information such as running processes, network connections, and encryption keys for forensic investigations.
Why is storage forensics important? Storage forensics plays a crucial role in digital investigations as it allows investigators to recover deleted or hidden files, track user activities, and uncover evidence related to cybercrimes or data breaches.
How is memory forensics useful? Memory forensics provides investigators with real-time insights into a system's current state, allowing them to identify active malware, detect malicious activities, and recover valuable artifacts that may not be present on disk.
What tools are commonly used in storage and memory forensics? Popular tools for storage forensics include EnCase, FTK Imager, and Autopsy. For memory forensics, tools like Volatility Framework, Rekall, and WinDbg are widely used.
